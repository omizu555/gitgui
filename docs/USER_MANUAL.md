## 1. 起動と初期画面

`git-gui.exe` を実行するとメインウィンドウが開きます。

- **初回起動時** — リポジトリ未登録のため「新しいタブ」ページが表示されます
- **2回目以降** — 前回開いていたプロジェクトタブが自動的に復元されます

起動時にすべてのタブの **未プル数**（↓N）と **変更ファイル数**（✎N）が自動取得されます。

---

## 2. リポジトリの追加

### 方法 A: ローカルフォルダを開く

1. 「新しいタブ」ページで **📂 既存のフォルダを追加** カードをクリック
2. フォルダ選択ダイアログでリポジトリのルートフォルダを選択
3. タブが追加され、ステータスが表示されます

### 方法 B: リポジトリをクローン

1. 「新しいタブ」ページで **📥 リポジトリをクローン** カードをクリック
2. リポジトリ URL を入力（HTTPS または SSH）
3. 保存先フォルダを選択
4. 「クローン開始」をクリック

### 方法 C: ドラッグ & ドロップ

- エクスプローラーから **フォルダ** をウィンドウ上にドロップ
- Git リポジトリであれば自動的にタブとして追加されます

### 最近閉じたプロジェクト

「新しいタブ」ページの下部に、最近閉じたプロジェクト（最大10件）が表示されます。クリックすると再追加できます。

---

## 3. タブの操作

ウィンドウ上部のタブバーで複数リポジトリを切り替えます。

| 操作 | 方法 |
|---|---|
| タブ切替 | タブをクリック |
| タブを閉じる | タブ右端の **×** ボタン |
| 新しいタブを追加 | タブバー右端の **+** ボタン |
| 並べ替え | タブをドラッグ & ドロップ |

### タブのバッジ表示

各タブには以下のバッジが表示されます（該当する場合のみ）:

- **✎N**（青色）— 変更されたファイル数（Staged + Unstaged）
- **↓N**（赤色）— 未プルのリモートコミット数

これらはタブ切替時も保持され、Fetch や自動フェッチのタイミングで更新されます。

---

## 4. ファイルの変更を確認する

左パネルの **変更ファイル** セクションに、Staged / Unstaged それぞれのファイルが表示されます。

- 各ファイルの右側にアイコンが表示されます:
  - **M** — 変更あり（Modified）
  - **A** — 新規追加（Added）
  - **D** — 削除（Deleted）
  - **R** — リネーム（Renamed）

### ファイルの選択

- **クリック** — 1ファイルを選択
- **Ctrl + クリック** — 複数ファイルを個別に選択
- **Shift + クリック** — 範囲選択

---

## 5. ステージとコミット

### ステージ

| 操作 | 方法 |
|---|---|
| 選択ファイルを Stage | 右クリック → **Stage**、または `Enter` キー |
| 全ファイルを Stage | 左パネルの **Stage All** ボタン |
| Stage を取り消す | 右クリック → **Unstage** |

### コミット

1. 左パネル上部のテキストエリアにコミットメッセージを入力
2. **✓ Commit** ボタンをクリック（または `Ctrl + Enter`）

### Amend（直前のコミットを修正）

1. **Amend（直前のコミットを修正）** チェックボックスを ON にする
2. 前回のコミットメッセージが自動入力されます
3. メッセージを編集して **✓ Commit** をクリック

---

## 6. リモート操作（Fetch / Pull / Push）

ツールバーのボタンから操作します。

| ボタン | 動作 |
|---|---|
| 🔄 **Fetch** | リモートの変更情報を取得（マージなし）。全タブのバッジが更新されます |
| ⬇️ **Pull** | リモートの変更を取得してマージ |
| ⬆️ **Push** | ローカルのコミットをリモートに送信 |

### ステータスバー

ウィンドウ下部に現在のブランチ名と ahead/behind 情報が表示されます:

```
main ↑2 ↓3
```
- **↑2** — 未 Push のローカルコミットが2件
- **↓3** — 未 Pull のリモートコミットが3件

---

## 7. ブランチ操作

### ブランチの切替

1. ツールバー右端の **ブランチ名（🔀 main ▼）** をクリック
2. ドロップダウンからブランチを選択
   - **ローカルブランチ** — そのまま切替
   - **リモートブランチ** — 自動でローカル追跡ブランチを作成して切替

### 新しいブランチの作成

1. ブランチドロップダウンの下部にある **＋ 新しいブランチを作成** をクリック
2. ブランチ名を入力して「作成して切り替え」をクリック

---

## 8. Stash（一時退避）

### Stash の保存

1. ツールバーの **📦 Stash ▼** をクリック
2. ドロップダウンの **+ New Stash** ボタンをクリック
3. 現在の変更がすべて退避されます

### Stash の復元

1. **📦 Stash ▼** をクリックして一覧を表示
2. 復元したい Stash の **Pop** ボタンをクリック
3. 変更が復元され、Stash エントリが削除されます

---

## 9. コミットログの確認と検索

右パネルにコミットログがグラフ付きで表示されます。

### グラフの見方

- **色付きの丸と線** — ブランチの流れを示します
- **ref バッジ** — ブランチ名やタグが色分けで表示されます:
  - 🔵 ローカルブランチ
  - 🟢 リモートブランチ
  - 🟡 HEAD
- **↓ 未プル**（赤バッジ）— フェッチ済みだがまだプルしていないコミット（行全体が半透明）

### ログの検索

ログ上部の検索バーで絞り込みできます。

| モード | 切替ボタン | 説明 |
|---|---|---|
| メッセージ検索 | 💬 | コミットメッセージまたは作者名で部分一致フィルタ |
| ファイル名検索 | 📄 | 指定パターンに一致するファイルを変更したコミットを検索 |

- モード切替: 検索バー右側のボタン（💬 / 📄）をクリック
- メッセージ検索はリアルタイムフィルタ、ファイル名検索はバックエンドで実行されます

### スクロール読み込み

ログは200件ずつ表示されます。下にスクロールすると自動的に追加読み込みされます。

## 10. Diff ビューア

1. 変更ファイルを選択して **右クリック → Diff を表示**（または `D` キー）
2. モーダルでハンク単位の差分が表示されます:
   - 🟢 緑背景 — 追加行
   - 🔴 赤背景 — 削除行
   - グレー — コンテキスト行（変更なし）
3. **×** ボタンまたは `Escape` キーで閉じます

---

## 11. コミット詳細

コミットログの行を **ダブルクリック** すると、コミット詳細モーダルが開きます。

表示内容:
- コミットハッシュ（全長）
- 作者名
- 日時
- 親コミット
- コミットメッセージ（全文）
- 変更ファイル一覧（ステータスアイコン付き）

---

## 12. コンテキストメニュー

変更ファイルを **右クリック** するとコンテキストメニューが表示されます。

| 項目 | ショートカット | 説明 |
|---|---|---|
| Stage | `Enter` | 選択ファイルをステージに追加 |
| Unstage | — | ステージから取り消し |
| Diff を表示 | `D` | Diff ビューアを開く |
| ファイルを開く | — | OS既定アプリでファイルを開く |
| パスをコピー | `Ctrl+C` | ファイルパスをクリップボードにコピー |
| Discard Changes | — | 変更を破棄（確認ダイアログあり） |

---

## 13. 設定

ツールバーの **⚙️** ボタンをクリックすると設定ダイアログが開きます。

### 自動フェッチ

| 設定項目 | 説明 |
|---|---|
| **自動フェッチ** トグル | ON にすると定期的にリモートの変更を確認します |
| **フェッチ間隔** | 5分 / 10分 / 15分 / 30分 / 60分 から選択 |

自動フェッチが実行されると:
- 全タブに対して `git fetch` が実行されます
- 各タブの未プルバッジ（↓N）と変更ファイルバッジ（✎N）が更新されます
- アクティブタブのコミットログも自動更新されます
- トースト通知「自動フェッチ完了」が表示されます

設定はアプリ終了後も保持されます（`%APPDATA%/GitGUI/app_data.json`）。

---

## 14. キーボードショートカット

| キー | 動作 |
|---|---|
| `Ctrl + Enter` | コミット |
| `Enter` | 選択ファイルを Stage |
| `D` | 選択ファイルの Diff を表示 |
| `Ctrl + C` | 選択ファイルのパスをコピー |
| `Escape` | モーダル / ドロップダウンを閉じる |

## 15. 外部ツール連携

ツールバーから以下の外部ツールを起動できます:

| ボタン | 動作 |
|---|---|
| 📂 **Explorer** | リポジトリフォルダをエクスプローラーで開く |
| 💻 **Terminal** | リポジトリフォルダでコマンドプロンプトを開く |
| 🌐 **Remote** | リモート URL をブラウザで開く（SSH URL は自動的に HTTPS に変換） |

---

## ウィンドウフォーカス自動更新

ウィンドウがフォーカスを受けた際（別のアプリから戻った時）、アクティブタブのステータスとコミットログが自動的に再取得されます。外部エディタでファイルを編集した後にウィンドウに戻ると、変更が即座に反映されます。

## トラブルシューティング

### Clone / Fetch / Pull / Push が認証エラーになる

GitGUI は認証が必要な操作にシステムの `git` コマンドを使用しています。以下を確認してください:

- `git credential-manager` が設定されているか
- SSH キーが `ssh-agent` に登録されているか
- コマンドラインから `git fetch` が正常に動作するか

### リモートボタンが無効（グレーアウト）になっている

リポジトリに `origin` リモートが設定されていない場合、Remote ボタンは無効になります。

### 変更が反映されない

ウィンドウをクリックしてフォーカスを戻すか、`Fetch` ボタンをクリックして強制更新してください。
